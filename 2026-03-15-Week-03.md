2️⃣ Key Concepts Learned

Application vs System Software
System software includes the operating system, drivers, and utilities that allow hardware and software to interact. Application software consists of programs designed to help users perform specific tasks such as browsing the internet, writing documents, sending emails, or collaborating with teams.

Productivity Software
Common productivity tools include word processors, spreadsheet applications, and presentation software. Platforms such as Microsoft Office, Google Workspace, and LibreOffice provide tools for creating documents, analyzing data, and presenting information.

Collaboration and Cloud Services
Modern work environments rely heavily on collaboration tools like Slack, Microsoft Teams, and Google Workspace. Cloud storage services such as Dropbox, Google Drive, and OneDrive allow users to store files remotely and collaborate in real time.

Software Licensing Models
Software is typically distributed under licensing agreements such as single-user licenses, site licenses for organizations, or subscription models. Users must accept the End User License Agreement (EULA), which defines how the software can legally be used.

Open Source vs Proprietary Software
Open-source software provides access to the source code and allows modification and redistribution, while proprietary software is owned by a company and distributed under strict licensing terms.

Software Distribution and Installation
Modern software is typically downloaded from official websites or app stores rather than physical media. Installation methods vary across operating systems, including executable installers on Windows, disk image files on macOS, and package managers in Linux.

3️⃣ Practical Work / Hands-On Practice

Although this module was primarily conceptual, I spent time reinforcing the concepts by reviewing different software ecosystems and installation processes across operating systems.

I also explored the troubleshooting framework used by CompTIA, which follows a six-step process:

1. Identify the problem
2. Establish a theory of probable cause
3. Test the theory
4. Establish a plan of action and implement the fix
5. Verify full system functionality
6. Document findings

Understanding this structured approach is important because troubleshooting is a core skill for both IT support and cybersecurity professionals.

5️⃣ Security Mindset Reflection

This week reinforced how closely software management and cybersecurity are connected.

Applications are one of the most common attack surfaces in any system. Vulnerabilities in software, outdated applications, or downloading programs from untrusted sources can introduce malware or other security risks.

Understanding how software is installed, updated, licensed, and maintained helps reduce these risks. It also highlights why patch management and regular system updates are critical in secure environments.

Even basic IT practices such as removing unused software, avoiding bloatware, and verifying software sources can significantly improve system security.

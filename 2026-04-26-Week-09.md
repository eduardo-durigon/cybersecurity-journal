## 🗓️ Cybersecurity Weekly Journal – Week 09

### 📚 What I Studied

This week I focused on **Linux fundamentals** and **basic Python scripting**, mainly around permissions, process management, and simple troubleshooting automation.

* Linux Users, Groups, and Permissions
* Linux Process Monitoring & Management
* Linux File Permissions & Ownership
* Python: System Check Assistant

---

### 🔑 Key Concepts

**Linux**

* Users and groups for access control
* File and directory permissions (rwx)
* Ownership and how it affects access
* Principle of least privilege
* Process monitoring (`ps`, `top`)
* Managing processes with PIDs and `kill`
* Permission limits between users

**Python**

* Variables and user input
* Conditional logic (`if`, `elif`, `else`)
* Basic string handling
* Simple decision-based scripts

---

### 🧪 What I Did

* Created multiple users and groups and assigned different access levels

* Built directories with restricted and shared access

* Used `chown` and `chmod` to control ownership and permissions

* Tested access between users to confirm restrictions were working

* Monitored running processes using `ps` and `top`

* Simulated background processes and managed them

* Tried to kill processes from different users to understand permission limits

* Practiced file permissions in both symbolic and numeric formats

* Ran into and solved permission errors like “Operation not permitted”

* Built a simple Python script that asks about system issues (disk space, internet) and gives basic recommendations

---

### 💡 Key Takeaways

* Linux security really comes down to how well users, groups, and permissions are set up
* Small mistakes in permissions can lead to security issues
* The principle of least privilege is something you actually apply, not just learn
* Process management is an important everyday skill in IT
* Permission errors are part of how Linux protects the system
* Even simple Python scripts can be useful for basic troubleshooting
